
Для начала про настройку .gitconfig
Вот команды настройки:
 - git config --global user.name<Вводите имя или ник латиницей в кавычках>
 - git config --global user.email <Вводите почту>
 
Теперь команда, которая превратит обычную папку в репо - git init

«Разгитить» папку, если что-то пошло не так:
Если вы случайно сделали Git-репозиторием не ту папку, её можно «разгитить». Для этого нужно удалить скрытую подпапку .git.
 - cd <папка с репозиторием> # перешли в папку
 - rm -rf .git # удалили подпапку .git 

Текущее состояние репозитория можно посмотреть с помощью команды git status

Чтобы Git понял, что файл нужен для сохранения в новой версии, вводим git add и затем через пробел название файла
 - git add --all # добавит все файлы к сохранению версии 

Сделать коммит можно командой git commit с ключом -m(от англ. message – сообщение), который присваивает коммиту название.

Ссылка на удаленный репо
 - git remote add origin git@github.com:<ВАШ НИКНЕЙМ>/first-project-git.git

Запушить 
 - git push -u origin main 

Клонировать проект 
 - git clone git@github.com:<ВАШ НИКНЕЙМ>/<ИМЯ ПРОЕКТА>.git
Когда появилось что-то новое 
 - git pull
